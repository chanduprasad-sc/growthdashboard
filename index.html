📖 Dashboard Guide

🚀 Getting Started
Data Loading: The dashboard loads data from dashboard_data.json (generated by preprocess.py) or from a live Google Sheets API endpoint. Data is automatically preprocessed to match calls with tickets, calculate occupancy rates, and detect repeat loops.

Date Range: Use the preset timeline buttons (Today, Last 7d, Last 30d, etc.) or set a custom date range. All tabs react to the selected date range.

Filters: Filter by Broker Family, Assigned POC, Support Agent, and Channel type. Use the "Include Care Emails" checkbox to optionally include email data in the Weekly Pulse view.
📑 Tab Guide
Weekly Pulse: Your primary weekly reporting view. Shows KPIs, POC heatmaps, charts, and an interactive attribute explorer. Every KPI card is clickable for deep-dive details.

Main Dashboard: Comprehensive channel overview with sub-tabs for Calls, WhatsApp, and Emails deep-dives. Each sub-tab provides detailed metrics, trends, and SLA compliance data.

Intelligence: Cross-channel insights including repeat contact loops, outlier detection, broker health scores, issue trending, and time-of-day patterns.

Monthly View: Date-wise aggregation table for any selected month. Shows daily breakdowns with a summary aggregate row at the bottom. Export to CSV supported.

Agent Performance: Per-agent scorecards showing calls handled, QA scores, SLA compliance, break time, and occupancy rates. Select a specific agent or compare all agents side-by-side.

Tickets & Chats: Browsable, searchable, paginated list of all support interactions. Filter by status and channel. Click any row to expand details. Every ID is hyperlinked to DevRev.

AI Summary: AI-powered narrative summary of your support operations using NVIDIA Nemotron. Generates actionable insights, top queries, and focus areas. Includes offline fallback.

Guide: You're reading it! 📖
🔍 Deep-Dive Interactions
KPI Cards: Click any KPI card on the Weekly Pulse tab to see the exact entries used to calculate that metric.

POC Cards: Click any POC block to open a full segment deep-dive showing branch friction, top RMs, and a raw interaction audit trail.

Table Rows: In Tickets & Chats, click any row to expand a detail panel showing full metadata, comments, QA scores, and recording links.

Monthly Dates: Click any date row in Monthly View to drill into all interactions for that specific day.

DevRev Links: All ticket IDs (TKT-xxxxx) and conversation IDs (CONV-xxxxx) are hyperlinked to open directly in the DevRev workspace.
🎨 Themes & Accessibility
Three Themes: Toggle between Light Mode (bright white), Dark Mode (space navy), and Black Mode (OLED-friendly pitch black) using the theme button in the sidebar footer.

Sidebar: Click the ◀ button to collapse the sidebar for more workspace. A floating ▶ button will appear to restore it.

Screenshots: Use the 📸 button to capture the current dashboard view as a PNG image.
📊 Data Sources
6 Data Sheets:
• Ozonetel Calls — Raw voice call records (UCID, duration, agent, status, recording URL)
• Ozonetel DevRev Tickets — Support tickets created from calls (matched via 20-second time window)
• WhatsApp Chats — RM conversations via WhatsApp (DevRev conversations)
• Care Emails — Email tickets from care@smallcase.com
• Ozonetel Agent Breaks — Agent break logs for occupancy calculation
• POC-Branch Mapping — Master mapping of internal POCs to partner branches
❓ FAQ
Q: Why are Care Emails excluded from Weekly Pulse by default?
A: Care Emails don't have branch or POC mapping, so they're optionally included via the checkbox to keep weekly reports focused on mapped partners.

Q: How is AHT calculated?
A: Average Handling Time is calculated from the Ozonetel Calls sheet "Duration" column for answered/connected voice calls, converted to seconds.

Q: How is occupancy calculated?
A: Occupancy = Talk Time / (Shift Time - Break Time) × 100, where shift time is 9 hours per active day.

Q: What defines a "repeat loop"?
A: When the same RM from the same broker/branch raises the same issue type within 7 consecutive days.

💡 How to get your Web App URL:

  1. In your Google Sheet, open Extensions ➔ Apps Script.
  2. Paste the contents of Codenew.gs and click save.
  3. Click Deploy ➔ New Deployment (Select type: Web app).

  5. Click Deploy, authorize permissions, and copy the generated Web App URL.
